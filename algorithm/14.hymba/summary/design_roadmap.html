Hymba 설계 로드맵 (Ablation Study)

Table 1: 아키텍처 변천사

Hymba는 한 번에 완성된 아키텍처가 아닙니다. 논문의 Table 1은 표준 트랜스포머에서 시작하여, 각 구성 요소를 단계적으로 추가하거나 변경하면서 성능과 효율성이 어떻게 변하는지를 보여주는 '설계 로드맵'을 제시합니다. 이 과정은 각 구성 요소가 Hymba의 최종 성능에 얼마나 기여하는지를 명확하게 보여주는 중요한 분석(Ablation Study)입니다.

모델 변경점 분석 및 해설
1. Transformer Baseline 표준 트랜스포머 아키텍처 모든 비교의 기준이 되는 모델입니다. 높은 성능을 보이지만 긴 시퀀스에서 효율성 문제가 발생합니다.
2. + Meta Tokens 입력 앞에 메타 토큰 추가 어텐션 싱크 문제를 해결하고, 모델이 입력을 처리하기 전에 일종의 '준비 운동'을 하도록 유도하여 안정성을 높입니다.
3. + Shared KV Cache 인접 레이어 간 KV 캐시 공유 추론 시 메모리 사용량을 크게 줄여줍니다. 모델이 더 일반화된 표현을 학습하도록 하는 정규화 효과도 있습니다.
4. + SWA 슬라이딩 윈도우 어텐션 적용 어텐션의 계산 복잡도를 $O(N^2)$에서 $O(N \times W)$로 낮춰, 긴 시퀀스에서의 계산 부담을 획기적으로 줄입니다.
5. Hymba (Final) SSM 헤드 병렬 추가 최종 형태입니다. 최적화된 어텐션과 효율적인 SSM을 병렬로 결합하여, 성능과 효율 두 마리 토끼를 모두 잡습니다.
